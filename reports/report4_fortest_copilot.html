Product Summary: Fortest is Fortude's intelligent test automation platform designed to streamline and accelerate functional, regression, integration, and data-validation testing across enterprise applications — especially Infor M3 and CloudSuite. Version 4.0.0 introduces AI-Powered Self-Healing and Microsoft Azure AI + Key Vault integration, pushing Fortest into the next generation of smart, resilient automation. This session analyses gaps in the current platform and maps how Copilot capabilities can address each one, ultimately moving Fortest from a niche execution tool to a competitive mid-tier Test Intelligence Platform.

New in Fortest 4.0.0
Two major new capabilities introduced in this release
🤖
AI-Powered Self-Healing
NEW v4.0
  • Uses machine learning models for intelligent locator analysis
  • Automatically detects changes in the UI/DOM and heals broken locators at runtime
  • Deep integration with Microsoft Azure AI for contextual understanding of UI changes
  • Reduces test flakiness and manual rework
  • Enhanced logs & reporting show which elements were healed and how

NEW v4.0

  • Supports standardized, centralized secrets management

04
What Infor Is Currently Not Doing
Native gaps in Infor's CloudSuite that create a persistent moat for Fortest
No Native Test Automation Tool
Infor provides no equivalent to Fortest's industry-specific M3 test libraries. TaaS is not a testing platform — it lacks test automation capabilities entirely.
No Self-Healing Automation
No AI-based locator healing or runtime intelligence. Infor provides no equivalent of Functionize or Testim's self-healing capabilities for CloudSuite screens.
No Visual Testing Solution
No equivalent of Applitools / visual regression for CloudSuite screens. Visual bugs that locator-based checks won't catch go undetected.
No Test Data Automation
Does not generate, mask, or orchestrate test data. Testers must manually manage test data sets, creating bottlenecks and compliance risks.
No TestOps Analytics
Does not offer dashboards to track flaky tests, heatmaps, coverage, or run analytics. No equivalent of ACCELQ, Tricentis LiveCompare, or Testim's analytics.
No NL-Driven Test Authoring
No equivalent of ACCELQ, Tricentis LiveCompare, or Testim for natural language test authoring. Test creation remains a specialist engineering activity.

06
How Copilot Closes the Gaps
Specific Copilot capabilities mapped to each Fortest gap — moving from a test execution tool to a Test Intelligence Platform
Visual AI / Visual Regression
Closes: AI Visual Testing Gap
What Copilot Can Do
  • Watch every visual checkpoint (screenshots, components, screens)
  • Detect visual differences (font/colour changes, missing buttons, layout shifts, pixel-level) and assess severity
  • Classify the type of change and explain it in simple language
  • Recommend action (Accept as new baseline, raise a defect) with side-by-side comparison UI
  • Track approvals for audit compliance
Fortude ActionProductize a Visual Assistant that watches failed runs, suggests visual diffs and severity. Bundle visual checks as an upsell SKU. Demo as a "Visual Guardrail" for regulated environments.
Governance & Audit for Self-Healing
Closes: Governance Gap
What Copilot Can Do
  • Explain every healing action in human language with confidence score
  • Show alternative options (Approve, Reject, Require Team Lead sign-off)
  • Apply enterprise governance policies defined by the customer
  • Create immutable audit trail (what changed, who approved, why it was suggested, timestamps)
  • Guide teams when adopting self-healing for the first time
Fortude ActionCopilot Governance Center presenting why changes were made, confidence scores, and alternative options. Keep immutable logs for audits. Target regulated organisations.
Low-Code / NL Test Authoring
Closes: Authoring Gap
What Copilot Can Do
  • Convert test scenarios described in plain English into structured test steps with validations, test data placeholders, and Fortest reusable functions
  • Generate test data automatically based on test steps, expected scenarios, and business constraints
  • Suggest valid supplier codes, SKUs, and customer names from M3 master data
Fortude ActionTest Authoring Assistant enabling business users (Product Owners, BAs) to describe scenarios and receive draft tests. Run pilot programs to measure scripting time reduction. Expand Fortest's buyer beyond technical QA.
Model-Based / Risk-Based Test Optimization
Closes: Suite Size Gap
What Copilot Can Do
  • Read release notes, commit messages, change descriptions
  • Identify which processes are affected (e.g. POs, inventory movements)
  • Map changes to relevant test cases using Fortest templates and historical patterns
  • Recommend a smaller, high-impact test suite (e.g. "Instead of 100 tests, run these 35 that cover all changed areas with 91% risk coverage")
  • Create an optimized test plan automatically with explained reasoning
Fortude ActionCopilot as Release Risk Advisor — begin with heuristic-based rules and progress to ML as telemetry improves. Position as premium Release Risk Analysis service.
TestOps Console
Closes: TestOps Gap
What Copilot Can Do
  • Summarize test health daily for QA managers and engineering leads
  • Pinpoint root cause patterns across failed test runs
  • Recommend operational actions (e.g. auto-create Jira tickets, mark flaky tests)
  • Predict issues before they become incidents ("this test has failed intermittently for 4 days")
Fortude ActionCopilot TestOps Console synthesizing trends and surfacing flaky tests. Market to engineering leaders and QA managers. Ensure clear audit trail and human override.
Security, Performance & Accessibility Hub
Closes: Quality Ecosystem Gap
What Copilot Can Do
  • Analyze patterns in test logs to identify flakiness — flag tests that fail intermittently
  • Suggest remediation (add wait, update locator, fix test data, etc.) and assign ownership
  • Aggregate outputs from SAST/DAST, load tests, and accessibility scanners
  • Synthesize business impact across functional, performance, security and accessibility results
Fortude ActionCopilot Quality Hub aggregating scanner outputs and synthesizing business impact. Bundle as a "Quality Suite" with connectors to popular security scanners.
What Copilot Will Not Solve Cannot replace domain expertise. Relies on telemetry — poor data equals poor recommendations. Cannot autonomously test without a strong underlying test model. Cannot rewrite complex automation architecture independently. These four limits define where Fortest still needs human expertise and engineering investment.

Recommended Roadmap
Three-horizon investment plan from closing essential gaps to durable differentiation
Short Term
Close Essential Market Gaps
Low to Medium Effort
  • Visual AI diff checks + rule-based baseline management
  • Lightweight visual compare using ML for layout/semantic differences
  • Human-in-the-loop Auto Heal Policies (reasoning per healed locator)

Mid Term
Empower Non-Technical Users
Medium Effort — Strategic
  • Low-Code / NL Test Authoring: create tests from plain English
  • AI snippets for assertions using LLMs with local model inference for privacy
  • Model-Based Coverage Mapping & Risk-Based Test Selection
  • Build process model to recommend minimal test suites based on change impact
  • Expand self-healing beyond UI to API + data layers
Long Term
Shift to Autonomous Decision-Making
Higher Effort — Durable Differentiation
  • Agentic Test Assistants: generate tests, triage failures, create bug reports, suggest fixes
  • Productize AI test agent for Infor M3 domain-specific automation
  • Autonomous regression suites that determine what to test
  • AI-led change impact modeling with automatic missing test case identification
  • Ecosystem & Marketplace: curated libraries, community templates
